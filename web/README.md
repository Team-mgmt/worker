# ShelfAlign AI - Web 📚

ShelfAlign AI는 2026 도서관 데이터 활용 공모전을 위해 개발된 **스마트폰 기반 도서관 서가 오배열 탐지 및 도서 탐색 서비스**입니다. 
이 레포지토리(web)는 사용자가 직접 상호작용하는 프론트엔드 UI와 관리자용 ShelfOps 대시보드를 포함하고 있으며, 영상 인식 및 데이터 처리를 담당하는 worker 서버와 연동되어 동작합니다.

---

## 🏗 전체 시스템 구조

ShelfAlign 서비스는 프론트엔드(`web`)와 백엔드/AI 파이프라인(`worker`) 두 개의 레포지토리로 분리되어 있습니다. 본 레포지토리는 사용자 접근을 담당하는 Web 영역입니다.

```text
[ Client (Web Repo) ]                    [ Server (Worker Repo) ]
+-------------------+                    +-----------------------+
|  사서/이용자 모바일 앱 |  --- 이미지/영상 --->  |  AI Pipeline (YOLO+OCR) |
|  ShelfOps 대시보드    |  <--- 분석/판정 결과 ---  |  도서관 장서 DB (PostgreSQL)|
+-------------------+                    +-----------------------+
```

서비스는 목적에 따라 두 가지 모드를 제공합니다:
- **사서 모드 (Librarian mode)**: 사서가 특정 서가 전체를 촬영하면, KDC 분류를 기반으로 오배열된 도서를 찾아냅니다.
- **이용자 모드 (Patron mode)**: 이용자가 OPAC에서 찾은 책이 실제 서가에 보이지 않을 때, 주변을 스캔하여 목표 도서가 존재하는지 보조 탐색을 수행합니다. (실시간 위치 추적이 아닌 단일 도서 매칭 보조 기능입니다)

## 📱 주요 화면 설명

### 1. 메인 화면
서비스의 진입점입니다. 사서 로그인 버튼과 이용자용 도서 탐색 시작 버튼이 제공되며, 이용 목적에 따라 모드를 선택할 수 있습니다.

### 2. 서가 촬영 화면 (사서용)
사서가 스마트폰 카메라로 서가를 훑듯이 촬영하는 화면입니다. 조명 상태 안내 및 인식률을 높이기 위한 촬영 각도 가이드라인 UI가 제공됩니다.

### 3. 인식 결과 화면 (사서용)
워커 서버에서 처리된 OCR 인식 도서 목록과 오배열 후보가 표시됩니다. 사서는 각 후보의 신뢰도(Confidence)를 확인하고 오배열 판정을 '확인' 또는 '기각'할 수 있습니다.

### 4. ShelfOps Dashboard (사서용 웹)
전체 서가의 점검 현황을 한눈에 파악할 수 있는 관리자 대시보드입니다. 점검 이력, 오배열 후보 수, 사서 확인율, 반복 오배열 발생 KDC 구간 히트맵, 그리고 점검 우선순위 추천 리스트를 제공합니다.

### 5. 도서관 설정 화면 🚧
(구현 중) 관리자가 소속 도서관 코드(`libCode`)를 입력하여 도서관 정보나루의 장서 DB와 연동하고, 동기화 상태를 확인하는 화면입니다.

### 6. 도서 탐색 화면 (이용자용)
이용자가 찾고 싶은 책의 정보를 설정하는 화면입니다. OPAC에서 미리 확인한 도서명, 저자, 청구기호, ISBN 중 하나 이상을 입력할 수 있습니다.

### 7. 서가 촬영 화면 (이용자용)
입력한 책을 찾기 위해 이용자가 서가 주변을 촬영하는 화면입니다. 사서용 화면과 유사하지만, 상단에 목표 도서 정보가 고정되어 표시됩니다.

### 8. 탐색 결과 화면 (이용자용)
촬영된 화면 내에서 목표 도서와 유사한 후보가 감지되었는지 안내합니다. 
- "유사 후보 감지됨 (위치: 화면 우측 하단)" 
- "현재 촬영 범위에서 후보를 찾지 못했습니다"
등의 메시지가 표시되며, ⚠️ 실시간 위치 추적이 아님을 명시하는 안내 문구가 포함됩니다.

## 🛠 기술 스택

- **Framework**: Next.js (또는 React 기반 프레임워크)
- **Styling**: Tailwind CSS, shadcn/ui 기반 커스텀 컴포넌트
- **State Management**: React Query / Zustand (예정)
- **Backend Communication**: REST API (fetch/axios)
- **Analytics**: 내부 이벤트 로깅 (향후 GA4 연동 예정)

## 🚀 로컬 실행 방법

본 프로젝트는 MVP 단계이며 지속적으로 업데이트되고 있습니다.

```bash
# 1. 의존성 설치
pnpm install

# 2. 로컬 개발 서버 실행
pnpm run dev
```

서버가 실행되면 브라우저에서 `http://localhost:5177/scan` (모바일 앱) 및 `http://localhost:5176` (대시보드)으로 접속할 수 있습니다.

---

## ⚙️ 환경 변수

프로젝트 최상단에 `.env` 파일을 생성하고 아래 변수들을 설정해야 합니다. (실제 키값은 보안상 레포지토리에 포함되지 않습니다)

```env
NEXT_PUBLIC_WORKER_BASE_URL=http://localhost:8000/api
NEXT_PUBLIC_FRONTEND_URL=http://localhost:5177
```

## 📂 레포지토리 구조

```text
/apps
  ├── /scan            # 사서/이용자 모바일 스캔 앱 (React)
  ├── /backoffice      # 관리자용 ShelfOps 대시보드
  └── /frontend        # 서비스 랜딩 및 소개 페이지
/packages
  ├── /client-common   # 공통 UI 컴포넌트 및 유틸리티
  └── /schema          # API 통신용 Zod 스키마
```
