# 도서관 장서·데이터 품질 지표 정의서 (v0.1)

> 목적: 여러 도서관을 **동일 기준으로 비교·평가**하기 위한 지표 정의.
> 범위: 정보나루 `itemSrch` (type=ALL) 응답만으로 산출 가능한 **장서(컬렉션) / 데이터 품질** 지표.
> "입지"는 여기서 **물리적 위치가 아니라 장서·데이터 측면의 우수성**으로 정의한다.
> 물리적 입지(인구·접근성·경쟁)는 별도 데이터가 필요하므로 이 문서 범위 밖이며, 후속 v0.2에서 다룬다.

## 0. 데이터 전제

- 단위: `itemSrch type=ALL` 의 한 레코드 = 한 장서(소장) 항목. `numFound` = 장서 규모.
- 수집 방식은 도서관 간 동일해야 한다: `type=ALL`, 동일 `pageSize`, 전체 페이지 수집.
- 사용 필드(실제 응답 확인됨):
  `isbn13`, `set_isbn13`, `bookname`, `authors`, `publisher`, `publication_year`,
  `class_no`(KDC), `class_nm`, `addition_symbol`, `vol`, `reg_date`,
  `callNumbers[].callNumber.{book_code, shelf_loc_code, shelf_loc_name, copy_code, separate_shelf_*}`
- 청구기호: 단일 문자열 필드는 없고 `class_no`(KDC) + `book_code`(도서기호=저자기호)로 구성한다. 예: `818 손73오`.

## 1. 규모 지표

### 1.1 장서 규모 (collection_size)
- 정의: 전체 장서 레코드 수 = `numFound`.
- 방향: 클수록 ↑ (단, 1인당/면적당 보정은 입지 데이터 필요 → v0.2).
- 비교 시 주의: 복본 포함 수치다. "고유 장서 수"는 1.2 참고.

### 1.2 고유 서지 수 (distinct_titles)
- 정의: 비공백 `isbn13` 기준 고유 ISBN 수 (+ ISBN 없는 레코드는 별도 카운트).
- 방향: 클수록 ↑ (장서의 실질 다양성).

## 2. 주제 균형·다양성 지표 (KDC 기반)

KDC 대분류 = `class_no` 첫 자리(0~9). 첫 글자가 숫자가 아니면 "미분류/비KDC".

### 2.1 KDC 대분류 커버리지 (kdc_coverage)
- 정의: 레코드가 1건 이상 존재하는 대분류 수 / 10.
- 방향: 클수록 ↑ (전 주제 균형 보유).

### 2.2 KDC 정규화 엔트로피 (kdc_norm_entropy)
- 정의: 대분류 분포의 Shannon 엔트로피를 log2(10)으로 나눈 값. 범위 0~1.
  - `H = -Σ p_i·log2(p_i)`, `norm = H / log2(10)`, p_i는 분류된 레코드 중 대분류 i 비중.
- 방향: 1에 가까울수록 ↑ (주제가 고르게 분포). 특정 주제 집중 컬렉션이면 낮아짐.

### 2.3 최대 편중도 (top_class_share) / 집중도 HHI
- 정의: `top_class_share` = 가장 비중 큰 대분류의 점유율. `HHI = Σ p_i²`.
- 방향: 낮을수록 ↑ (특정 주제 쏠림이 적음). 단 전문/특화 도서관은 의도적으로 높을 수 있어 해석 주의.

## 3. 데이터 품질·정합성 지표 (비교용 핵심)

도서관마다 메타데이터 입력 충실도가 달라, 비교의 신뢰도·매칭 가능성을 좌우한다.

### 3.1 ISBN 결측률 (isbn_missing_rate)
- 정의: `isbn13`가 빈 값인 레코드 비율.
- 방향: 낮을수록 ↑ (외부 데이터 결합·중복 판정·매칭에 유리).

### 3.2 KDC(class_no) 결측률 (classno_missing_rate)
- 정의: `class_no`가 빈 값인 비율.
- 방향: 낮을수록 ↑.

### 3.3 저자기호(도서기호) 결측률 (bookcode_missing_rate)
- 정의: `callNumbers[0].callNumber.book_code`가 빈 값인 비율.
- 방향: 낮을수록 ↑.
- 주의: itemSrch는 청구기호를 단일 필드로 주지 않지만 `book_code`(도서기호) 자체는 제공한다.
  따라서 이 결측률은 "저자기호 자체가 비어있는" 순수 결측만 측정한다.

### 3.4 청구기호 구성 가능률 (callnumber_constructible_rate)
- 정의: `class_no`와 `book_code`가 **모두** 있어 `class_no + book_code` 청구기호를 만들 수 있는 비율.
  = 1 − (둘 중 하나라도 결측인 비율).
- 방향: 높을수록 ↑ (서가 위치 추정/VLM 매칭에 직접 기여).

## 4. 장서 운영 성향 지표

### 4.1 복본 비율 (dup_copy_rate)
- 정의: 비공백 `isbn13` 기준 `1 − (고유 ISBN 수 / ISBN 보유 레코드 수)`.
- 해석: 높으면 인기서 다부 보유(수요 대응형), 낮으면 다양성 우선형. 단순 우열이 아니라 **운영 성향**으로 해석.

### 4.2 신간 비율 (recent_share, 보조지표)
- 정의: `publication_year`가 최근 N년(기본 5년) 이내인 레코드 비율. (`reg_date` 기반 보조 산출 가능.)
- 방향: 높을수록 ↑ (장서 신선도). 단 데이터 품질(연도 결측·오기)에 민감 → 보조지표로만.

## 5. 비교·정규화 프로토콜

- 모든 지표는 도서관별로 동일 수집 조건에서 산출한다.
- 비율 지표(결측률·복본률 등)는 이미 0~1 스케일이라 직접 비교 가능.
- 규모 지표(1.1, 1.2)는 절대값 차가 크므로 비교 시 **로그 스케일 또는 분위(percentile) 정규화** 후 사용.
- 종합 점수가 필요하면(향후): 각 지표를 방향에 맞춰 0~1로 정규화 → 가중합.
  - 예시 초안 가중치(데이터 품질 강조): 데이터정합성 0.4, 주제다양성 0.3, 규모 0.2, 운영성향 0.1.
  - 가중치는 분석 목적(예: VLM 서가 매칭 적합 도서관 선별)에 따라 조정한다.

## 6. 산출물 매핑

`scripts/eda_nowon.py` 가 산출하는 값과 본 지표의 대응:

| 지표 | 산출 컬럼 |
|---|---|
| 장서 규모 | `collection_size` |
| 고유 ISBN / 보유 | `distinct_isbn`, `records_with_isbn` |
| KDC 커버리지 | `kdc_classes_present` |
| KDC 정규화 엔트로피 | `kdc_norm_entropy` |
| ISBN 결측률 | `isbn_missing_rate_pct` |
| KDC 결측률 | `classno_missing_rate_pct` |
| 저자기호 결측률 | `bookcode_missing_rate_pct` |
| 청구기호 구성불가율 | `callnumber_missing_rate_pct` (구성가능률 = 100 − 이 값) |
| 복본 비율 | `dup_copy_rate_isbn_pct` |

> 신간 비율(4.2), 고유 서지 수의 ISBN-없는 보정, HHI는 현재 스크립트 미산출 → 필요 시 추가.

## 7. 실측에서 확인된 비교 주의사항 (노원 111058, 2026-06 수집)

1. **ISBN 결측률은 도서관 비교 지표로 신뢰도 낮음.** 노원 표본에서 ISBN 결측률이 전 등록연도 구간(≤2015 포함) 0.00%로 나왔다. 이는 `itemSrch`가 **ISBN 보유 자료만 반환**하기 때문으로 추정된다. 따라서 이 API로는 "ISBN 없는 자료"가 애초에 표본에 안 들어와 결측률이 구조적으로 0에 수렴한다. → 데이터 품질 비교에는 부적합. 참고용으로만.
2. **저자기호(book_code) 결측률은 등록 시기에 강하게 의존.** 노원: ≤2015년 18.95% → 2016~2020 6.02% → 2021~2026 2.36%. 따라서 **수집 표본의 등록연도 구성이 결측률을 좌우**한다. 도서관 간 공정 비교를 하려면 (a) 전수, 또는 (b) 등록연도 분포를 맞춘 표본/층화 추정이 필요하다. "최신순 앞 N페이지" 표본은 결측률을 과소추정한다.
3. **장서 규모는 표본이 아니라 `numFound`(API 메타)로 측정.** 페이지네이션 합계는 한도 초과 시 빈 응답으로 과소 집계될 수 있으므로 `numFound`를 정본으로 쓴다.
4. **수집 한도.** IP 미등록 인증키는 1일 500 호출 제한(`errCode: outOflimit`). 전수 비교가 필요하면 IP 등록 또는 pageSize를 키워 호출 수를 줄인다(예: pageSize 500 → 약 485 호출).
