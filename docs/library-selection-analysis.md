# 서울 공공도서관 ShelfAlign 도입 적합도 분석

## 1. 분석 목적

ShelfAlign MVP 대상 도서관을 데이터 기반으로 선정하기 위해, 서울 공공도서관을 대상으로 도입 적합도를 정량 평가한다.

## 2. 데이터 출처

정보나루(data4library.kr) — 국립중앙도서관 공공 도서 데이터 API 2종 사용.

| API | 용도 |
|---|---|
| `libSrch` | 서울 공공도서관 전체 목록 + 장서수(BookCount) 수집 |
| `itemSrch` | 도서관별 장서 샘플 수집 → 저자기호 결측률·KDC 분포 측정 |

## 3. 후보 도서관 선정

서울 공공도서관 355개 중 아래 조건으로 필터링:

- **장서 15만권 이상** — 오배열이 의미 있게 발생하는 최소 규모
- **어린이·영어특화·청소년·교육청 도서관 제외** — ShelfAlign 대상이 일반 종합자료실

→ 20개 후보 선정, 최종 **16개 데이터 확보** (4개는 API 오류로 미수집)

## 4. 샘플 수집 방법

각 도서관당 `itemSrch type=ALL`로 **150건(3페이지 × 50건)** 샘플 수집.

- `type=ALL`: 등록일 무관 전체 장서 대상 (최신 등록순 정렬)
- 150건 표본으로 비율 추정 — 오차 ±8% 수준

## 5. 측정 지표

### 5-1. 장서 규모 (BookCount)
libSrch API 반환값. 오배열 발생 가능성 및 이용자 규모의 proxy.

### 5-2. 저자기호(도서기호) 결측률
`itemSrch` 응답의 `callNumbers[0].callNumber.book_code`가 비어있는 비율.

ShelfAlign 매칭 엔진(`matching_service.py`)의 점수 계산식에서 직접 사용:

```
청구기호 매칭 경로 = KDC점수 × 0.55 + 저자기호점수 × 0.45  (가중치 0.65)
저자기호 없으면 → 제목/저자 fuzzy fallback으로 격하 (가중치 0.35)
```

저자기호 결측이 많을수록 매칭 정확도 하락.

### 5-3. 한국문학(810-819) 비중
`class_no` 앞 2자리가 "81"인 레코드 비율. 현재 MVP 범위(810-819 종합자료실)에서 데모 시나리오의 풍부도를 나타냄.

## 6. 점수 계산 공식

각 지표를 0~1로 정규화 후 가중합:

```
size_score   = (장서수 - 최솟값) / (최댓값 - 최솟값)
bcode_score  = 1 - (저자기호결측률 / 100)
korlit_score = (한국문학비중 - 최솟값) / (최댓값 - 최솟값)

종합점수 = 0.40 × size_score
         + 0.35 × bcode_score
         + 0.25 × korlit_score
```

**가중치 근거:**

| 지표 | 가중치 | 근거 |
|---|---|---|
| 장서 규모 | 40% | 오배열 발생 빈도·서비스 가치의 핵심 |
| 저자기호 결측률 | 35% | 매칭 엔진 작동 여부에 직결 |
| 한국문학 비중 | 25% | 현재 MVP 범위 기준 (전체 KDC 확장 시 비중 조정 필요) |

## 8. 해석

**상위 3개관 추천 근거:**

- **서울도서관(1위)**: 장서 61만으로 압도적 규모. 서울 대표 도서관으로 공모전 임팩트 최대. 저자기호 결측 8%는 16개 중 유일하게 0%가 아니나 규모 우위로 1위.
- **광진정보도서관(2위)**: 결측 2%·한국문학 37%로 균형 최상. 40만 장서.
- **성동구립도서관(3위)**: 결측 0%·44만 장서. 데이터 품질 완벽.

**제외 권고:**

- 성북정보도서관(저자기호 결측 54%), 도봉문화정보도서관(42%) → 매칭 엔진 정상 작동 불가

**현재 MVP 노원중앙(13위):**

데이터 품질(결측 0%)은 우수하나 장서 규모에서 상위 3개관 대비 경쟁력 낮음.
