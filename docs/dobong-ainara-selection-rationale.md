**지하보존서고(직원문의)** = 17,399건(16.6%)은 직원 전용 접근 구역으로, 이용자가 직접 꺼내 보는 공개 서가가 아니라는 점에 유의 (오배열 위험의 성격이 다른 구역과 다름 — §3 참고).

### 2.5 발견된 이상치
1. `classNoNum`이 KDC 유효범위(0~999.x)를 벗어난 값 **8건** (예: `1400.001`, `8843.0`, `3813.8`) — 분류기호 오기입 추정
2. `publicationYear` — 미래 연도(2027년 이후) **3건**, 1900년 이전(0년 포함) **47건** — 데이터 입력 오류
3. `copyCode` — 95.5%가 결측이지만 book당 holding 최댓값(33)과 copyCode 최댓값(33)이 일치 → 복본 자체는 실재하나 `copyCode` 필드 신뢰도는 낮음(대부분 미기입)

## 3. 선정 후보 지표 (오배열 탐지 서비스 필요성 근거)

기존 `docs/library-quality-indicators.md`는 "장서·데이터 품질" 비교 목적으로 만들어진 지표 체계다. 본 절은 **"오배열 탐지 서비스가 왜 필요한가"** 관점으로 후보 지표를 재구성한다.

| 지표 | 도봉아이나라 값 | 오배열 위험과의 연결 논리 |
|---|---|---|
| 서가 구역 복잡도 (구역 수 × 층위) | 11구역, 3개 층위(지하/1/2층) | 구역이 세분화될수록 이용자가 반납/browsing 후 엉뚱한 구역에 책을 꽂을 확률 증가 |
| 주제 편중도 (최다 KDC 비중) | 문학 56.9% | 같은 대분류(비슷한 표지·색)의 책이 밀집 → 육안 구분이 어려워 사람이 오배열을 놓치기 쉬움, VLM 매칭의 효용이 커짐 |
| 복본 비율 | 6.15% | 같은 책이 여러 권 있으면 특정 권이 다른 자리에 있어도 "빠졌다"는 사실 자체를 알아채기 어려움 |
| 청구기호 원본 데이터 신뢰도 | 원본 컬럼 98.01% 결측 → 재계산 의존 | 도서관 자체 메타데이터 정리가 약함을 시사 → 서가 실측 검증(=본 서비스)의 필요성을 뒷받침 |
| 유아 전용 구역 비중 | 22,897건(21.88%) | 유아자료실·유아 디지털실이 소장량의 약 5분의 1 — 아동 이용자는 책을 정확한 자리에 되돌려놓을 확률이 낮다는 것이 일반적 통념(정량적 인과 근거는 아니며 참고 수준), 그 대상 구역 비중이 크다는 것 자체는 실측 |
| 장서 규모 | 홀딩 104,653건 (API 최신 112,081건) | 규모가 클수록 수작업 전수 점검이 비현실적 → 자동화 서비스 필요성 증가 |

## 4. 정정 사항

1. 이전 `docs/dobong-ainara-catalog-load-report.md`에 기록된 "복본(같은 책 2권 이상) 12,854건(전체 책 종수의 약 12%)"은 검증 없이 기록된 오류였다. 본 문서(§2.2)에서 isbn13 기준·book_id 기준 두 방식으로 재계산한 결과 **6,015종(6.15%)**로 확인되어, 이 수치를 정본으로 한다.
2. 본 문서 §2.4의 서가 위치 목록은 최초 작성 시 터미널 인코딩 깨짐으로 일부 구역명을 잘못 추정 기재했었다(예: 실제 "유아자료실(1층)"을 "종합자료실(1층)"으로, "잉글리쉬존(2층)"을 "학습마루터(2층)"으로 오기). UTF-8로 재추출하여 정정했으며, 그 결과 유아 전용 구역 비중(21.88%)이라는 더 강한 근거가 드러났다.
