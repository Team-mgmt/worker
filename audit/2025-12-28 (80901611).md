  1. CODE DUPLICATION - Must Consolidate

  2.1 CRITICAL: Duplicated _prepare_image Method

  Location 1: worker/worker/scan.py:74-91
  Location 2: worker/processors/base.py:40-53

  Both methods are nearly identical:
  async def _prepare_image(self, image_id, image_key, image_type):
      extension = pathlib.Path(image_key).suffix
      filepath = f"/tmp/shelfalign-worker/{image_type}/{image_id}{extension}"

      if not os.path.exists(filepath):
          response = await self.client.get_object(...)
          async with aiofiles.open(filepath, "wb") as f:
              await f.write(await response["Body"].read())

      # Only difference: ScanWorker version calls self._disk_monitor.record_access(filepath)

      image = cv2.imread(filepath)
      # ... same error handling and color conversion

  Impact: Maintenance burden - fixes must be applied in two places.

  Action: Extract to a shared utility function or have BaseProcessor use ScanWorker's implementation via composition.

  2.2 CRITICAL: Duplicated Version Detection Logic

  Location 1: worker/worker/scan.py:93-128 (_detect_version)
  Location 2: worker/processors/base.py:55-94 (_detect_datamatrix)

  Both do the same work:

  1. Call read_barcodes() for DataMatrix
  2. Validate exactly 4 codes found
  3. Parse position and version from each
  4. Validate all positions present
  5. Validate all versions match

  The only difference is that _detect_datamatrix also collects annotation data.

  Impact: Bug fixes must be applied twice; risk of divergence.

  Action: Consolidate into single implementation. ScanWorker._detect_version could call BaseProcessor._detect_datamatrix and just return the version.

  1. REDUNDANCY - Should Simplify

  3.1 Redundant Constant: POSITIONAL_QR_DATA

  Evidence: worker/consts.py
  POSITIONS: list[Literal["LT", "RT", "RB", "LB"]] = ["LT", "RT", "RB", "LB"]  # Line 6
  POSITIONAL_QR_DATA: list[Literal["LT", "RT", "RB", "LB"]] = ["LT", "RT", "RB", "LB"]  # Line 8 - IDENTICAL

  Both are identical lists. POSITIONAL_QR_DATA is used in exactly one place (base.py:182).

  Action: Remove POSITIONAL_QR_DATA and use POSITIONS everywhere.

  3.2 Redundant Image Thresholding

  Evidence: worker/processors/v1.py - threshold/close operations computed twice:

  First occurrence: Lines 38-41 in _process_child_areas():
  gray = cv2.cvtColor(image, cv2.COLOR_RGB2GRAY)
  thresh = cv2.adaptiveThreshold(gray, 255, ...)
  k = cv2.getStructuringElement(cv2.MORPH_RECT, (5, 5))
  close = cv2.morphologyEx(thresh, cv2.MORPH_CLOSE, k)

  Second occurrence: Lines 168-171 in process():
  gray = cv2.cvtColor(image, cv2.COLOR_RGB2GRAY)
  thresh = cv2.adaptiveThreshold(gray, 255, ...)
  k = cv2.getStructuringElement(cv2.MORPH_RECT, (5, 5))
  close = cv2.morphologyEx(thresh, cv2.MORPH_CLOSE, k)

  Impact: Wasted CPU cycles processing the same image twice.

  Action: Compute threshold/close once in process() and pass to _process_child_areas().
